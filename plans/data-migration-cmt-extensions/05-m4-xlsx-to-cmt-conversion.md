# Milestone 4 ‚Äî Staging Excel ‚Üí CMT Package Conversion

## Goal

Convert a populated staging Excel workbook into a complete CMT-compatible data package (data.xml + data_schema.xml + sidecars + manifest), ready for import via `txc data package import`.

## CLI Command

```
txc data package convert --input migration.xlsx --schema data_schema.xml -o ./package/ [--keys data_keys.xml] [--on-error warn|fail]
```

### Arguments

| Argument      | Description                                                         |
|---------------|---------------------------------------------------------------------|
| `--input`     | Path to the populated staging Excel workbook                        |
| `--schema`    | Path to `data_schema.xml` (generated by M1)                        |
| `-o`          | Output directory for the package                                    |
| `--keys`      | Optional path to `data_keys.xml` for alternate key resolution       |
| `--on-error`  | Error handling mode: `warn` (log and continue) or `fail` (abort)    |

## Output

A complete package directory:

```

‚îú‚îÄ‚îÄ data.xml
‚îú‚îÄ‚îÄ data_schema.xml
‚îú‚îÄ‚îÄ [Content_Types].xml

```

## Conversion Pipeline

The conversion follows a strict sequential pipeline:

### 1. Read

- Open the workbook with `StagingExcelReader` (from M2c).
- Parse all entity sheets (migrated only ‚Äî `include_in_cmt = true`).
- Parse optionset sheets, M:N sheets, BPF sheet, API sheets.

### 2. Validate

- Verify all required columns present per entity.
- Check data types match expected schema types.
- Flag empty required fields.
- Collect validation issues (warnings or errors based on `--on-error`).

### 3. Resolve Optionset Composites

Convert composite "Label [value]" back to integer values:

```
"Active [1]"  ‚Üí  1
"Won [3]"     ‚Üí  3
```

Parse the value from between the square brackets. Validate that the extracted value exists in the schema's optionset definition.

### 4. Resolve Lookup Composites

Convert composite "Name [GUID]" back to GUID entity references:

```
"Contoso Ltd [a1b2c3d4-...]"  ‚Üí  a1b2c3d4-...
```

Parse the GUID from between the square brackets. Validate that the GUID format is correct.

### 5. Synthesize Deterministic GUIDs

For new records that don't have a GUID in column A, generate a **deterministic GUID** using the alternate key values (from `data_keys.xml`). This ensures:

- The same record always gets the same GUID across repeated conversions.
- Cross-references between entities resolve correctly.
- Re-imports are idempotent (same GUID = upsert, not duplicate).

Algorithm: MD5 hash of `entityname|keyfield1=value1|keyfield2=value2` ‚Äî matching the existing `txc` HTTP endpoint for key resolution.

### 6. Emit `data.xml`

Write the CMT-compatible data XML with all records, using the resolved values (integers for optionsets, GUIDs for lookups).

### 7. Emit Sidecars

Generate sidecar files from the Excel data:

| Source                     | Sidecar                |
|----------------------------|------------------------|
| `ownerid` columns          | `data_owners.xml`      |
| `statecode` columns        | `data_state.xml`       |
| BPF Advancement sheet      | `data_bpf.xml`         |
| API sheets                 | `data_actions.xml`     |
| Caller ID data             | `data_callerid.xml`    |

### 8. Emit Manifest

Write `txc-package.xml` listing all generated sidecars and the execution order in `data_postimport.xml`.

### 9. Validation Report

Output a summary of:

- Records processed per entity
- Warnings (type coercion issues, missing optional fields)
- Errors (if `--on-error fail`, these would have aborted earlier)
- Checksum mismatches (see below)

## Type Coercion Table

| Schema type | Cell raw | Emitted XML `value` | Notes |
|---|---|---|---|
| `string` | `"abc"` | `abc` | Trim trailing whitespace optional. |
| `int` | `42` / `42.0` | `42` | Reject `4.5`. |
| `decimal`/`money`/`float` | `1.23` | `1.23` | Use invariant culture. |
| `bool` | `Yes`/`No`/`true`/`false`/`1`/`0` | `true`/`false` | Case-insensitive. |
| `datetime` | Excel date / ISO string | ISO 8601 UTC | Honor schema `dateMode`. |
| `picklist` / `state` / `status` | Composite "Label [value]" or numeric | numeric value | Resolve via optionset sheet. |
| `multiselectpicklist` | `"label1, label2"` | `value1;value2` | Sentinel `-1` is forbidden ‚Äî error. |
| `entityreference` / `customer` / `owner` | Composite "Name [GUID]" | GUID | Alt-key references rewritten via M5 `KeyAwareXmlRewriter` when `--keys` is provided. |
| `guid` | text | text (validated) | Regex `^[0-9a-f-]{36}$`. |

## Checksum Verification

Row checksums (column B in the Excel) are **recomputed** during conversion and compared against the stored values:

- **Match**: Row is clean ‚Äî no unexpected modifications.
- **Mismatch**: Row was modified outside of normal editing flow. A warning is emitted (or error if `--on-error fail`).

This enables delta detection: if checksums match, the row hasn't changed since the template was generated or last converted.

## Replaces Existing Command

This replaces the existing naive `DataPackageConvertCliCommand`. The old `--no-schema` flag is **deprecated for one release cycle** (emits a deprecation warning, then removed in the following release).

## Implementation

New type: `XlsxToCmtConverter` in `TALXIS.CLI.Features.Data/Xlsx/`

The converter orchestrates the full pipeline, delegating to:

- `StagingExcelReader` (M2c) for reading
- `PackageManifest` + sidecar writers (M3) for output
- `IRecordKeyService` for deterministic GUID synthesis
- `SchemaXmlWriter` (M1) for schema copy
